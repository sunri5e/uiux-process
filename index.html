Our process

RESEARCH

It all starts with research. At this stage we investigate business domain, all available requirements, variety of target audience, context of usage etc in order to get as much functional and nonfunctional requirements as possible.

DO:

  • Requirements analysis
  • User research (stakeholders interview, observation, personal interviews, forms and surveys etc.)
  • Competitors research
  • Designing Personas

DELIVER:

  • Mindmaps
  • User Journey maps
  • User Flows
  • Personas
  • User stories
  • etc.

INFORMATION ARCHITECTURE

Now it's time to define what kind of data or rather content will be delivered to user, which options user will be provided with in order to work with data, in which way a data will be visualized.

DO:

  • Navigation (global, local, context, ...)
  • Search
  • Data Hierarchy

DELIVER:

  • Content strategy documentation
  • Site maps
  • Application flows
  • ER diagrams

PROTOTYPING

Prototyping is responsible for providing an imitation of a real product.

DELIVERABLES:

  • Prototypes of various fidelity

DESIGN

The logical last step of enwrapping of the outcomes of all previous stages into fancy and pleasant visual identity. This is an important part since according to studies users tent to provide higher allegiance to visually appealing products.

DELIVERABLES:

  • UI concept with visual identity.
  • Set of graphical files with UI designs.

TEST AND ITERATE

The UX design is an iterative process. That's why after Wireframing and Prototyping we check whether the proposed solution solves user's problems. And iterate-iterate-iterate.

DELIVERABLES:

  • Usability testings outcomes
  • UX Assessment
